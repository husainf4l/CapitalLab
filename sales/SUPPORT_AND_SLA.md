## Priority Definitions

| Priority | Definition | Example |
|---|---|---|
| **P1 — Critical** | System completely unavailable. No patients can be processed. | Frontend/API down; database unreachable |
| **P2 — High** | Core workflow severely impaired. Some workaround exists. | Cannot enter results; PDF generation failing |
| **P3 — Medium** | Feature not working but workaround available. | Notification not sending; report formatting issue |
| **P4 — Low** | Minor visual or usability issue. No workflow impact. | Typo, color issue, report margin slightly off |

## Escalation Process

### Level 1 — First Response
- Support ticket or WhatsApp message acknowledged
- Initial triage: classify priority, assign engineer

### Level 2 — Technical Escalation
- Triggered if: P1 not resolved within 2 hours, or P2 not resolved within 8 hours
- Senior engineer assigned
- Client receives status update every 30 minutes for P1

### Level 3 — Management Escalation
- Triggered if: P1 not resolved within 4 hours
- Capital Lab CTO/Director notified
- Direct communication line to client lab director

## What Is and Is Not Covered

### Covered Under Support SLA
- Software bugs (documented, reproducible behavior inconsistent with specifications)
- Performance issues caused by Capital Lab software
- Configuration issues caused by Capital Lab deployment
- Security patches and updates
- Integration issues with Capital Lab supported formats (HL7, ASTM, CSV)

### Not Covered Under Support SLA
- Issues caused by client's hardware (printers, barcode scanners, client devices)
- Network connectivity issues on the client side
- Third-party service failures (WhatsApp API outage, SMS provider downtime)
- Issues caused by unauthorized modifications to the software
- Training requests (covered separately under Training packages)
- Custom development requests (quoted separately)

## How to Submit a Support Request

### Option 1 — Support Portal (All Tiers)
1. Go to: support.capitallab.io (or email support@capitallab.io)
2. Click "Submit Ticket"
3. Select priority level
4. Describe the issue with:
   - What you were trying to do
   - What happened instead
   - Screenshot (if applicable)
   - User account and branch
   - Time the issue occurred

### Option 2 — WhatsApp (Business & Enterprise)
1. Send a message to the dedicated support WhatsApp group
2. Include: issue description, urgency, screenshot if possible
3. A support engineer will respond within the SLA window

### Option 3 — Phone (Enterprise Only)
- Call the dedicated on-call number for P1 issues
- Available 24/7 under Enterprise plan

## Uptime SLA (Enterprise Only)

Capital Lab guarantees **99.9% monthly uptime** for clients on the Enterprise plan.

| Monthly Downtime | Uptime % |
|---|---|
| < 43.8 minutes | 99.9% — SLA met |
| 43.8 min – 4.4 hours | 99.9% – 99.0% — Service credit issued |
| > 4.4 hours | < 99.0% — Enhanced credit + root cause report |

**Service Credits:**
- 99.0% – 99.9%: 5% of monthly subscription as credit
- < 99.0%: 10% of monthly subscription as credit

*Planned maintenance windows are excluded from uptime calculation.*
